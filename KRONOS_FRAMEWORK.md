## 一、核心理念

> **不是预测机器，是执行系统。**

交易的核心不是"预测对方向"，而是"用系统化的方法执行已验证的规律，并在执行中不断学习"。

**铁律：**
- ✅ 批量回测、统计概率、扫描结构性失效、自动化执行
- ❌ 解读新闻叙事、判断市场情绪、预测黑天鹅、主观干预交易

## 二、系统架构（五层）

```
┌─────────────────────────────────────────────────────────┐
│  Layer 1: 市场环境检测 (Market Regime Detector)         │
│  - Bull / Bear / Range 三态识别                         │
│  - EMA20/50/200 均线结构                                │
│  - 输出：当前环境 + 策略选择指引                         │
├─────────────────────────────────────────────────────────┤
│  Layer 2: 因子有效性监控 (IC Monitor)                  │
│  - 每小时计算 RSI/Vol/ADX 因子的IC值                   │
│  - IC>0.05 因子有效；IC>0.10 强有效                    │
│  - 输出：有效因子列表 + 置信度                           │
├─────────────────────────────────────────────────────────┤
│  Layer 3: 信号引擎 (Signal Engine)                      │
│  - 基于有效因子生成交易信号                              │
│  - 多周期确认：日线定方向 → 4H确认 → 1H入场           │
│  - 2h冷却去重机制                                       │
│  输出：买入/卖出/观望 + 置信度分数                       │
├─────────────────────────────────────────────────────────┤
│  Layer 4: 风险管理 (Risk Manager)                       │
│  - 仓位计算：置信度% × 最大仓位%                       │
│  - 止损：ATR动态止损（1.5xATR）                         │
│  - 移动止盈：3xATR触发保本                             │
│  - 熔断：单日亏损>5% → 停止交易                        │
│  - 强制平仓：24h持Hold强制出场                          │
├─────────────────────────────────────────────────────────┤
│  Layer 5: 执行层 (Execution Layer)                     │
│  - OKX API 自动下单                                     │
│  - 滑点监控：记录实际成交价vs信号价                      │
│  - 交易日志：每笔记录 timestamp/价格/滑点/手续费        │
└─────────────────────────────────────────────────────────┘
```

## 三、市场环境检测规则

### 三态定义

| 状态 | 条件 | 可用策略 |
|------|------|---------|
| **Bull** | EMA20 > EMA50 > EMA200 | 现货做多 + ETH BB双向 |
| **Bear** | EMA20 < EMA50 < EMA200 | BTC RSI双向 + 观望 |
| **Range** | MA20纠缠，走势不明 | 观望 or BTC RSI超买超卖 |

### 策略环境匹配表

| 环境 | BTC RSI现货 | ETH BB合约 | AVAX突破 | DOT RSI |
|------|------------|-----------|---------|---------|
| Bull | ✅ | ✅ 双向 | ✅ | ✅ |
| Bear | ✅ 双向 | ❌ | ❌ | ✅ |
| Range | ✅ 超买超卖 | ❌ | ❌ | ❌ |

## 五、四层验证体系

```
L1: 历史回测
    标准：全周期(2020-2026)正收益
    工具：backtest_engine.py
    
L2: Walk-Forward（步行前向）
    标准：≥10窗口，100%正收益
    工具：walkforward_v2.py
    
L3: 极端行情测试
    标准：≥60%窗口正，最差>-30%
    重点：2022熊市（含FTX）窗口必须正收益
    工具：bear_market_test.py
    
L4: 蒙特卡洛模拟
    标准：1000次模拟，5%分位数>-50%，中位数>0
    工具：batch_monte_carlo.py
    
缺一不可。L2/L3/L4未通过 → 禁止实盘。
```

## 六、风险管理规范

### 仓位规则

| 币种 | 最大仓位 | 说明 |
|------|---------|------|
| ETH | 总资金10% | 中置信度策略 |
| BTC | 总资金10% | 中置信度策略 |
| AVAX | 总资金5% | 波动大，突破系统 |
| DOT | 总资金15% | 高置信度策略 |

**总仓位上限：45%（熊市降到20%）**

### 止损规则

| 策略类型 | 止损方式 |
|---------|---------|
| RSI策略 | 2% 固定 或 1.5xATR（取大者） |
| 突破策略 | 3xATR |
| 趋势跟踪 | 3xATR + 移动止盈 |

### 移动止盈规则
1. 盈利达到 3xATR → 移动止损到 **保本**
2. 24h强制出场（防止钝化）

### 熔断规则

| 条件 | 操作 |
|------|------|
| 单日亏损 > 5% | 停止交易1天 |
| 单日亏损 > 10% | 停止交易3天 |
| 最大回撤 > 30% | 全仓停止交易 |

## 七、因子有效性监控（IC系统）

### IC计算公式
```
IC = Spearman相关系数(因子值, 未来收益)
```

### IC阈值

| IC值 | 含义 | 操作 |
|------|------|------|
| IC < 0.02 | 无效 | 不生成信号 |
| IC 0.02-0.05 | 弱有效 | 降低仓位50% |
| IC 0.05-0.10 | 有效 | 正常仓位 |
| IC > 0.10 | 强有效 | 全额仓位 |

### IC历史缓存
- 文件：`~/.hermes/cron/output/ic_history_cache.json`
- 每日09:00更新
- 衰减警告：IC从高点下降>0.015 → 飞书预警

## 十三、关键认知总结

### 最重要的发现

1. **BTC是唯一的**：其他币种不能用同样策略
   - BTC RSI<35胜率57%，ETH 47%，SOL 39%
   
2. **趋势跟随 > 均值回归**（在BTC上）
   - BTC 2018崩盘：持有-66%，趋势跟随+4371%
   - 10年数据：趋势跟随PF=5.8，年化+127%

3. **四层验证缺一不可**：
   - 历史回测 → Walk-Forward → 极端行情 → 蒙特卡洛
   - 通过L3才能上实盘

4. **BTC做空是送钱**：
   - 牛市做空BTC胜率30%，亏损61%
   - BTC只做多，ETH可以双向

5. **杠杆是一把双刃剑**：
   - 1x：日常使用
   - 2x：确定性高时（熊市BTC RSI双向）
   - 3x以上：风险过高，禁止
